1. 작업 커밋시 권장사항.
  - 하나의 커밋에 한 단위의 작업을 넣도록 한다. 
    (아이디 찾기 개발 후 커밋, 비번 찾기 개발 후 커밋. 이후 머지)
  - 커밋 메시지는 어떤 작업이 이뤄졌는지 알아볼 수 있도록 작성

2. 커밋 메시지 컨벤션
  - 널리 사용되는 커밋 메시지의 작성방식

  type: 제목

  body 
   ...
   ...
   ...

   footer (참고 내용)

   ex)
   feat: 압축파일 미리보기 기능 추가🐛
   
   사용자의 편의를 위해 압축을 풀기전에 다음과 같이 압축파일을 미리보기 할 수 있도록 함. 
    - 마우스 우클릭
    - 윈도우 탐색기 또는 맥 파인더의 미리보기 창

   Close #125(에러 코드) 

3. Type의 종류
  - feat  : 새로운 기능 추가시
  - fix   : 버그 수정
  - docs  : 문서 수정
  - style : 공백, 세미콜론, 스타일 수정 등
  - repactor : 코드 리팩토링
  - perf : 성능개선 
  - test : 테스트 추가 .
  -
